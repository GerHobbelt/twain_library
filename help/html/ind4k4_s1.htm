DTWAIN Is Not...

 Top 
•        An image display library.  There are no functions internal to DTWAIN that displays or prints Device Independent Bitmaps (DIBs).  The only DIB related functions are the internal read / write routines that DTWAIN uses a DIB needs to be saved to a file, and the DIB's that are generated when a device acquires an image.  There are no functions to rotate, skew, crop, etc any images.  Many imaging libraries (both free and commercial) include functions that manipulate DIBs, so it is highly recommended that you use one of these libraries with DTWAIN for a complete package.

 

•        A library that should be used for solely for implementing a TWAIN Source   If a programmer wishes to write a TWAIN Source, the programmer needs to know and use the TWAIN specification in its 'rawest' form, and not use DTWAIN to do the job.  However, you can write an application with DTWAIN that can thoroughly test a Source.  For example the TWCAP program makes extensive tests on any Source that has been installed.
